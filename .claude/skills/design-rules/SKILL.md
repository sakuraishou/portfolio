# デザイン指針（トップページ基準・シンプル第一）

新しい見た目を作るときの基準。**トップページ（`src/app/(frontend)/page.tsx` の各 Section）が唯一の参照**。新規パーツもこの世界観から外れないようにする。実装規約は `/scss-rules` を併用すること。

## 大原則

1. **極力シンプル**。要素・色・装飾は足すより削る。1 画面で主役は 1 つ。
2. **トップページに馴染ませる**。既存の配色・余白・角丸・フォント運用をそのまま流用する。新しいトークンを増やさない。
3. **温かみ + 余白**。オレンジ×クリームの暖色で、余白をたっぷり取って軽やかに見せる。
4. 迷ったら `About` / `Skills` セクションを手本にコピーして崩す。
5. **SP（モバイル）を主戦場として作り込む**。採用担当・閲覧者の多くはスマホで見る。PC の縮小版で済ませない（詳細は下記「SP（モバイル）を作り込む」）。

## SP（モバイル）を作り込む（重要）

PC を作ってから SP を「とりあえず縮めただけ」にしない。**SP こそ実際に一番見られる画面**。SP 単体で破綻なく、意図して気持ちよく仕上げる。実装したら必ず自分で SP 表示を確認する。

最低限チェックする:

- **改行位置**：見出し・キャッチが不格好な位置で折り返していないか。意図した位置で切るなら `<br className="sp" />`（SP のみ改行）、PC では改行を消す。
- **文字サイズ・行間**：小さすぎ／詰まりすぎないか。本文は最低 `1.4rem`〜、行間 `1.7`〜。
- **余白**：間延び・詰まりを避ける。固定ヘッダー分の上余白を確保（`7rem`〜目安）。
- **タップ領域**：リンク・ボタンは指で押せる大きさ（`4.4rem` ＝ 約 44px 四方以上）。
- **横はみ出し**：横スクロールを出さない（長い英単語・コードは折返し、`overflow` を確認）。
- **要素の崩れ**：横並びが窮屈なら縦積み（`flex-direction: column`）に。カード・表は全幅で。
- **画像**：`width: 100%` ＋ 適切な比率。巨大化・潰れがないか。
- **確認幅**：実機相当の **320px / 375px** で破綻しないこと。

モバイルファーストで組む（SP を基準に `@include responsive(md/lg)` で広げる。実装詳細は `/scss-rules`）。

## カラーパレット（`_styles/_variables.scss`）

これ以外の色を増やさない。新色が要ると思ったら、まず既存で表現できないか考える。

| 変数 | 値 | 役割 |
|------|-----|------|
| `$main` | `#f78c00` | メインのオレンジ。アクセント・見出し英字・強調 |
| `$accent` | `#ffa836` | 明るいオレンジ。背景の塗り分け（例: About 背景） |
| `$bg` | `#ffeedf` | クリーム。基本背景・淡いチップ背景 |
| `$bg2` | `#e4e2e1` | グレー。補助的な区切り背景 |
| `$fontBase` | `#5e3100` | 焦げ茶。本文・見出しの文字色 |
| `#fff` | — | 反転（濃い背景上の文字）。白黒は直書き可 |

- 基本は「クリーム背景 × 焦げ茶文字」、強調に「オレンジ」。これだけで成立させる。
- 色付き背景セクション（例: About のオレンジ）では、本文は読みやすさ優先で**焦げ茶のまま**、見出し（`UI/Title`）だけ白に反転して階層を立てる。背景が暗い場合は本文も `#fff` に反転する。

## タイポグラフィ

- **本文・日本語**: `$font-main`（Zen Kaku Gothic New）。本文 `1.6rem`〜、行間 `1.5`〜`1.9`。
- **見出しの英字（上品）**: `@include display`（Fraunces）。ヒーローの大見出し・名前など。※旧 Dancing Script の置き換え。
- **ラベル・データの英字（等幅）**: `@include mono`（IBM Plex Mono）。セクション英語ラベル・ナビ・型番・数値・ステータス・`/manual` 全般。**英字ラベルは基本これ**（旧 Quicksand を廃止し mono に統一）。
- 日本語見出しは太字 `700`、英字ラベルはやや細め。サイズは SP→PC で 1.2〜1.5 倍に上げる。

## 反復して使う「型」

新しいセクションでもこの型を踏襲すると一発で馴染む。

- **セクション見出し**: `UI/Title` を使う。独自の見出し装飾は作らない。2 系統:
  - **スペックシート版（トップ各セクションはこちら）**: `<Title en="ABOUT" no="01">私について</Title>`。mono の連番ラベル（`01 / ABOUT`）＋ヘアライン＋背面に Fraunces の英字ウォーターマーク。色は CSS 変数（`--title-accent` / `--title-fg` / `--title-wm`）で受けるので、色付き背景のセクションは className 側で上書きする（例: `.about .aboutTitle { --title-accent: #fff; ... }` と特異度を上げて反転）。
  - **従来版**: `no` 無し。日本語見出し＋背面に mono（IBM Plex Mono）のウォーターマーク（`$main` / `opacity: 0.2`）。`/manual` の表紙などで使用。
- **スペックシートの語彙**: トップ各セクションは「桜井 翔という"製品"の仕様書」コンセプト。mono の小ラベル（`MODEL` / `背景 / BG` 等）・key-value 行・ヘアライン・連番（01/02…）・ステータス点で"エンジニアらしい構造"を出す（MV / About / Skills 参照）。`/manual`（取扱説明書）と地続き。装飾系の英字ラベルは小さめ、見せたい本文・キーワードは `1.6rem`〜を確保。
- **コンテナ**: 横幅は `.wrap`（`max-width: 120rem` 中央寄せ）に乗せる。トップ各セクションは原則 `.wrap` で左端を揃える。
- **セクション余白**: `<section>` 既定の縦パディング（SP `8rem` / PC `16rem`）に従い、詰め込みすぎない。
- **角丸**: `border-radius` は `0.8rem`〜`1rem` 程度のやわらかい角。鋭角は使わない。
- **チップ/タグ**: クリーム背景（`$bg`）+ オレンジ文字（`$main`）の丸みのある小要素（About のタグ参照）。

## レイアウト / モーション

- **モバイルファースト**。SP を基準に組み、`@include responsive(md)` で広げる。**ブレークポイントは `md`（768〜）に統一**し、`lg` 等の中間段階は原則使わない（`/scss-rules` 参照）。
- 大きめレイアウトは `grid-template-areas` で意図を明示（About 参照）。
- 動きは**控えめ**に。スクロール誘導や軽いホバー程度。過剰なアニメーションでシンプルさを壊さない。
  - 動的な値は CSS 変数で渡す（例: Header の `style={{ '--delay': ... }}`）。

## やらないこと（シンプルを守るため）

- パレット外の色・グラデーション多用・濃い影の多用。
- セクションごとにバラバラの見出しスタイルを作ること（`UI/Title` に統一）。
- 情報の詰め込み。余白を削ってまで要素を増やさない。
- フォントの新規追加（原則）。日本語 = Zen Kaku、英字 = Fraunces（見出し）/ IBM Plex Mono（ラベル・データ）の 3 書体で完結させる。4 つ目を足す前に、既存の役割で表現できないか必ず考える。
