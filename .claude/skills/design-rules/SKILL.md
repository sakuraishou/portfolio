# デザイン指針（トップページ基準・シンプル第一）

新しい見た目を作るときの基準。**トップページ（`src/app/(frontend)/page.tsx` の各 Section）が唯一の参照**。新規パーツもこの世界観から外れないようにする。実装規約は `/scss-rules` を併用すること。

## 大原則

1. **極力シンプル**。要素・色・装飾は足すより削る。1 画面で主役は 1 つ。
2. **トップページに馴染ませる**。既存の配色・余白・角丸・フォント運用をそのまま流用する。新しいトークンを増やさない。
3. **温かみ + 余白**。オレンジ×クリームの暖色で、余白をたっぷり取って軽やかに見せる。
4. 迷ったら `About` / `Skills` セクションを手本にコピーして崩す。

## カラーパレット（`_styles/_variables.scss`）

これ以外の色を増やさない。新色が要ると思ったら、まず既存で表現できないか考える。

| 変数 | 値 | 役割 |
|------|-----|------|
| `$main` | `#f78c00` | メインのオレンジ。アクセント・見出し英字・強調 |
| `$accent` | `#ffa836` | 明るいオレンジ。背景の塗り分け（例: About 背景） |
| `$bg` | `#ffeedf` | クリーム。基本背景・淡いチップ背景 |
| `$bg2` | `#e4e2e1` | グレー。補助的な区切り背景 |
| `$fontBase` | `#5e3100` | 焦げ茶。本文・見出しの文字色 |
| `#fff` | — | 反転（濃い背景上の文字）。白黒は直書き可 |

- 基本は「クリーム背景 × 焦げ茶文字」、強調に「オレンジ」。これだけで成立させる。
- 濃い背景セクション（`$accent` 塗り）では文字を `#fff` に反転する。

## タイポグラフィ

- **本文・見出し**: `$font-main`（Zen Kaku Gothic New）。本文 `1.6rem`〜、行間 `1.5`〜`1.9`。
- **装飾の英字（筆記体）**: `@include dancing`（Dancing Script）。名前ロゴ等の遊び要素に少量だけ。
- **ラベル英字（モダン）**: `@include quicksand`（Quicksand）。セクション英語ラベルなど。
- 日本語見出しは太字 `700`、英字ラベルはやや細め。サイズは SP→PC で 1.2〜1.5 倍に上げる。

## 反復して使う「型」

新しいセクションでもこの型を踏襲すると一発で馴染む。

- **セクション見出し**: `UI/Title` を使う。`<Title en="ABOUT">私について</Title>` のように、日本語見出しの背面に大きな英語ウォーターマーク（`$main` / `opacity: 0.2`）が透ける。独自の見出し装飾を作らない。
- **コンテナ**: 横幅は `.wrap`（`max-width: 120rem` 中央寄せ）に乗せる。
- **セクション余白**: `<section>` 既定の縦パディング（SP `8rem` / PC `16rem`）に従い、詰め込みすぎない。
- **角丸**: `border-radius` は `0.8rem`〜`1rem` 程度のやわらかい角。鋭角は使わない。
- **チップ/タグ**: クリーム背景（`$bg`）+ オレンジ文字（`$main`）の丸みのある小要素（About のタグ参照）。

## レイアウト / モーション

- **モバイルファースト**。SP を基準に組み、`@include responsive(md/lg)` で広げる（`/scss-rules` 参照）。
- 大きめレイアウトは `grid-template-areas` で意図を明示（About 参照）。
- 動きは**控えめ**に。スクロール誘導や軽いホバー程度。過剰なアニメーションでシンプルさを壊さない。
  - 動的な値は CSS 変数で渡す（例: Header の `style={{ '--delay': ... }}`）。

## やらないこと（シンプルを守るため）

- パレット外の色・グラデーション多用・濃い影の多用。
- セクションごとにバラバラの見出しスタイルを作ること（`UI/Title` に統一）。
- 情報の詰め込み。余白を削ってまで要素を増やさない。
- 新しいフォントファミリーの追加。
