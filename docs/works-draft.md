### 説明文（一覧用・短い紹介） ✅

```
複数ドメインの Google Search Console / Analytics のデータを毎週自動で集計し、AI（Claude）が前週比などを分析して週報にまとめる社内ツール。ドメインごとに手作業で見て回っていたレポート業務を自動化しました。Nuxt（Vue）× GAS × Claude CLI で開発。
```

### 課題・背景 ✅

```
弊社では複数のドメイン（サイト）を運用しており、各サイトの Google Search Console / Analytics を毎週ドメインごとに見て回るのは時間がかかり、現実的に厳しい状態でした。

「数字を集める・見る・比べる」という手間を自動化し、人は判断や改善の議論に集中できるようにするために、週報を自動で生成するツールを開発しました。
```


### 工夫した点・詰まった所 ✅

```
・複数ドメインの一括処理：ドメインごとにバラバラだった確認作業を、まとめて自動集計・自動分析する流れに集約し、運用の手間を大きく削減しました。

・GASによる定期取得：毎週のデータ取得を Google Apps Script で自動化し、スプレッドシートに蓄積。前週比・任意期間比較の土台にしています。

・AI分析のコスト最適化：分析に Claude を使いつつ、API課金を避けるため Claude CLI をローカルで実行。「AIは使いたいが運用コストは抑えたい」を両立させました。

・比較で「変化」が分かる週報：蓄積データがあれば前週との比較、指定があれば任意データとの比較を行い、ただの数字の羅列ではなく増減や傾向が読み取れる週報にしています。
```

### 結果・学び ✅

```
これまでドメインごとに手作業で行っていた週報づくりを自動化し、確認・集計にかかっていた時間を大幅に削減できました。

GAS・スプレッドシート・AI（Claude CLI）を組み合わせ、「コストを抑えつつ実務に効くAI活用」を実プロダクトとして形にできた実績です。
```


### 課題・背景 ✅

```
自社では複数ドメインのWebサイトをWordPressで運用しており、なかでも「エリアページ（例：松戸市）」を重要な集客面と位置づけています。各エリアページには、表示する式場の情報をリレーションで紐づけています。

ただ運用面では、「今このドメインのこのエリアに、どの式場が表示されているか」を確認するだけでも各ページを見に行く必要があり、変更するにはWordPressにログインして操作する手間がありました。さらに「あのエリアの式場の表示順を変えてほしい」といった細かな依頼が現場から日常的に発生し、その都度の対応が負担になっていました。これを一括で扱える仕組みとして開発しました。
```

### 技術選定・設計判断（← 見せ場） ✅

```
フロントは Next.js（React）。②のコールセンター案件と同じくReact系で開発しています。

既存のWordPress資産はそのまま活かしつつ、WordPress REST API を通じて各ドメインの掲載状況を取得。「どのエリアに、何が、どの順番で表示されているか」を一画面で横断的に把握できるようにしました。WordPressを作り替えるのではなく、APIで“上に乗せる”形にすることで、既存サイトを止めずに運用を改善できる構成にしています。
```

### 工夫した点・詰まった所 ✅

```
・横断的な可視化：複数ドメインにまたがる「エリアページ × 式場」の掲載状況を、ページを一つずつ見て回らなくても一画面で確認できるようにしました。

・非エンジニアでも操作できるUI：WordPressにログインせず、画面上で式場の表示順の変更や追加ができる設計に。専門知識がなくても、現場の依頼をその場で反映できます。

・“詳しい人”のボトルネック解消：これまではWordPressを操作できる人を介する必要がありましたが、上層部や担当者が直接操作できるようにすることで、依頼→対応の往復をなくしました。

・WP REST API連携：既存のWordPress運用を壊さず、APIを介して掲載情報を読み書きする構成にしました。
```

### 結果・学び ✅
